# Обновление изображений для категории "Соки"

## Проблема
Все изображения соков используют placeholder URL (via.placeholder.com) вместо реальных изображений товаров.

## Решение

### Вариант 1: Автоматический парсинг (Рекомендуется)

Создан скрипт `scripts/parse-juice-images.ts` который автоматически парсит реальные URL изображений.

**Запуск:**
```bash
# Установить зависимости если нужно
npm install

# Запустить скрипт
npx tsx scripts/parse-juice-images.ts
```

**Что делает скрипт:**
1. Читает файлы с данными соков (krasnoeibeloe-juices-part1.ts, krasnoeibeloe-juices-part2.ts)
2. Для каждого товара открывает sourceUrl
3. Парсит HTML и находит изображение (Open Graph или основное фото товара)
4. Заменяет placeholder URL на реальный
5. Сохраняет обновленные файлы

### Вариант 2: Ручное обновление

Если автоматический парсинг не работает, можно обновить вручную:

1. Откройте файл `client/src/lib/krasnoeibeloe-juices-part1.ts`
2. Для каждого товара:
   - Откройте `sourceUrl` в браузере
   - Найдите изображение товара (правый клик → Копировать адрес изображения)
   - Замените `imageUrl` на скопированный URL

**Пример:**
```typescript
// Было:
{ name: "Нектар мультифрукт 1.93л", 
  imageUrl: "https://via.placeholder.com/100x150/FF8C00/FFFFFF?text=Nektar" }

// Стало:
{ name: "Нектар мультифрукт 1.93л", 
  imageUrl: "https://krasnoeibeloe.ru/upload/iblock/abc/product.jpg" }
```

### Вариант 3: Использование CDN изображений

Если на сайте krasnoeibeloe.ru есть паттерн URL изображений, можно использовать его:

**Типичные паттерны:**
- `https://krasnoeibeloe.ru/upload/iblock/[ID]/product.jpg`
- `https://cdn.krasnoeibeloe.ru/products/[SKU].jpg`

### Вариант 4: Fallback на SVG placeholder

Текущая реализация уже использует SVG fallback:
```typescript
const imageUrl = ingredient.imageUrl || getIngredientImage(ingredient);
```

Это означает что если `imageUrl` отсутствует или не загружается, будет показан красивый SVG placeholder с цветом ингредиента.

## После обновления

1. **Обновить данные в БД:**
```bash
npx tsx scripts/init-extended-ingredients.ts
```

2. **Перезагрузить страницу конструктора** - изображения должны отображаться

## Проверка результата

1. Откройте конструктор коктейлей
2. Выберите категорию "Соки"
3. Все 43 товара должны отображаться с изображениями слева
4. Изображения должны быть реальными фото товаров, не placeholder

## Дополнительно

Если нужно обновить изображения для других категорий (газировка, миксеры и т.д.), используйте тот же подход.
